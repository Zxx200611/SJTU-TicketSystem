### 存储部分：

其中的所有日期存储方式为 int 表示距离2026年6月1日的天数。其中的所有时间存储方式为 int 表示距离当天早上00:00的分钟数。

核心类：

+ 用户（class User)：BplusTree\<User\>Users
  + 【第一关键字】用户名（username)：string(20)。
  + 密码（passwd）：string(30)。
  + 姓名（chinese_name)：string(5*3=15)。
  + 邮箱（mail）：string(30)。
  + 权限（privilege）：int。
+ 车次（class Train）：BPlusTree\<Train\>Trains
  + 【第一关键字】标识符（train_id）：string(20)。
  + 途径站数（station_num）：int。
  + 途径站（stations）：string(3*10=30)[100]。
  + 余票数（seats）：int\[100][100]。表示第 $i$ 天第 $j$ 站到第 $j+1$ 站的余票数。
  + 票价（prices）：int[100]。表示第 $i$ 站到第 $i+1$ 站的票价。
  + 发车时间（departure）：int(<24*60=1440)。
  + 行车时间（travel_times)：int[100]。
  + 停留时间（stop_times)：int[100]。
  + 起售日期（sale_start）：int。
  + 终止日期（sale_end）：int。
  + 列车类型（type）：char。
  + 是否发布（is_released）：bool。
+ 订单（Order）：BPlusTree\<Order\>Orders
  + 【第一关键字】用户名（username）：string(20)。
  + 【第二关键字】交易时间戳（tag）：int。
  + 车次标识符（train_id）：string(20)。
  + 起点（s_station），终点（t_station）：string(30)。
  + **发车日期**（depar_date），出发日期（s_date），到达日期（t_date）：int。
  + 出发时间（s_time），到达时间（t_time）：int。
  + 总价格（price）：int。
  + 购票数量（count）：int。
  + 状态（status）：
    + 0：pending。
    + 1：success。
    + 2：refunded。

辅助类：

+ StationTimeTrain：BPlusTree\<..\> Arriv,Depar
  + 【第一关键字】车站名称（station）：string(30)。
  + 【第二关键字】日期（date）：int。
  + 【第三关键字】时间（time）：int。
  + 车次标识符（train_id）：string。
  + 发车日期（start_date），站编号（sta_id），累计时长（s_time），累计价格（s_price）。
+ SemiOrder：BPlusTree<..> Pend
  + 【第一关键字】车次标识符：train_id。
  + 【第二关键字】**发车日期**（depar_date）：int。
  + 【第三关键字】交易时间戳（tag）。
  + 用户名（username）。


B+树：

+ Users：存储用户信息。
+ Trains：存储车次信息。
+ Stations：存储车站名称。
+ Arriv,Depar,Pend：见上。

其他：

+ 登录列表（login_list）：map(string,bool). login_list[username]=1 表示此用户登录。

### 指令部分：

+ 用户部分
  + add_user：往Users中添加用户。
  + login：往 login_list[username]=1。
  + logout：在 login_list[username]=0。
  + query_profile：在Users中查找。
  + modify_profile：删除原User，加入新的。
  
+ 火车部分
  + add_train：在Trains中添加。枚举沿途车站并在Arriv,Depar中插入。
  
  + delete_train：在Trains中删除。枚举沿途车站并在Arriv,Depar中删除。
  
  + release_train：在Trains中修改。
  
  + query_train：计算并输出信息。
  
  + query_ticket：在 Depar 中查询所有在日期 date 从 s 出发的记录，在 Arriv 中查询所有在日期 [date,date+3] 到达 t 的记录。分别按照 train_id 排序，双指针求出 train_id 匹配且出发日期相同的对。
  
  + query_transfer：枚举所有在日期 date 从 $s$ 出发的记录，求出：对于每个可用的车次，s 在其中是第几站，累计价格，以及累计时间。然后对所有在日期 date 后方经过 $t$ 的车次求出 $t$ 在其中是第几站，累计价格以及累计时间。
  
    然后枚举中转站，遍历中转站日期 [date,date+3] 的 Arriv 记录，筛选出曾经经过 s 且发车日期为 date 的那些，记为集合 $A$。遍历中转站日期 date 后方的 Depar 记录，筛选出将要经过 $t$ 的那些，记为集合 $B$。将 $A$ 和 $B$ 分别按照时间排序。
  
    求出 $B$ 的后缀“到达时间/价格 最小、次小者”，然后双指针。
  
+ 订单部分
  + buy_ticket：
    + 若可行，输出并修改Trains，Orders。
    + 若不可行且允许候补，则修改Pend，Orders。
  + refund_ticket：
    + 修改Trains，Orders。
    + 根据退票车次的 id 和发车日期在 Pend 中查找候补记录，依次判断是否候补成功。
  
+ 其他
  + clean：清除数据。
  + exit：退出程序。
