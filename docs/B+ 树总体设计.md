## B+ 树总体设计

### 存储

仅一个文件，文件头为一个 int 表示树根位置。

紧接着为每个结点的数据：

+ 标号（int）：自己在文件中的位置。
+ 儿子个数（int）：若为叶节点则代表值个数。
+ 儿子位置数组（int[M+1]）：每个儿子结点在文件中的位置，若为叶节点则弃用。
+ 儿子最大值数组（value_pair[M+1]）：每个儿子结点的最大值，若为叶节点则代表存储的每个值。
+ 后继指针（int）：指向比他大的下一个叶子，若非叶节点则弃用。

每个 value_pair 包含一个 string(64) 和一个 int。

### 操作

树中有一个始终存在且唯一的最大值哨兵。其 key 为 ~~~~...~，value 为 INT_MAX。

+ 初始化：
  + 若文件中已经存在则读取。
  + 否则初始化：
    + 创建文件头，哨兵（根）。
  
+ 插入：函数参数为当前查找的结点编号。返回值为一个value_pair表示向父节点上传的值。
  + 递归往下找到最大值大于等于它的第一个叶子，插入。
  + 若当前结点大小为 M+1：
    + 则分裂为左右两个结点 L,R，其中 L 的编号直接继承，R 为新结点。
    + 若为叶子（后继指针大于0），则修改 L 和 R 的后继指针。
    + 重新计算 L 的最大值并返回。
  + 否则返回一个key为空的东西，表示不需要上传。
  
+ 删除：函数参数为当前查找的结点编号，以及它左右任意一个兄弟的编号。
  + 递归往下找到最大值大于等于它的第一个叶子，删除。
    + 若删除了当前叶子中的最大值，需要更改到根路径上的所有索引。（实现思考中）
  
  + 若当前结点非根且大小严格小于 (M+1)/2：
  
    + 若兄弟的大小加本结点大小小于等于 M：合并，在父节点中删除一个索引。
    + 若兄弟的大小大于(M+1)/2：借一个关键字。修改到根路径上的所有索引。
  
  + 若根非叶子且大小为1：删除。
